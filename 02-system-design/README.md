## Estrutura de um Design Doc

1. Cabeçalhos (autores, revisores, status, data)
2. Visão Geral (1-2 parágrafos)
3. Escopo e Contexto
4. Objetivos e Fora de Escopo
5. O Design (solução proposta + trade-offs)
6. Alternativas consideradas
7. Preocupações Transversais
8. Testabilidade, Plano de Implantação, Perguntas em aberto

## Ferramentas para Design Docs

- Confluence, Google Docs, GitHub (Markdown)
- PlantUML, Structurizr, Web Sequence (diagramas)
