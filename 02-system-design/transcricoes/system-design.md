# System Design

System design é o processo de definição de arquitetura, componentes, módulos, interface e dados para atender os requisitos especificados.

**Design Requirements:**
- Move Data
- Store Data
- Transform Data

Points that make design choices better than the others:
- **Availability** = (uptime / uptime + downtime) — quanto maior, melhor

## Importância

- Pensar na arquitetura de forma intencional
- Racionaliza as definições que realmente importam
- Explorar possíveis soluções
- Nos ajuda a ter uma visão de presente e futuro do software
- Exercita a forma de pensar e planejar diferentes tipos de solução de software

## System Design vs Big Techs

- Normalmente faz parte da maioria dos processos seletivos em Big Techs
- Forma do candidato pensar
- Repertório e o nível de profundidade nas tecnologias
- Capacidade de dedução
- Capacidade de comunicação
- Capacidade de ser confrontado

## System Design no Mundo Real

- System design é uma ferramenta na mão do arquiteto de soluções
- É uma forma do arquiteto se expressar
- Gerar convicções
- Vender
- Especificar e documentar

## Técnica e Metodologia — 6 Elements

1. **Requisitos** — Entendê-los, entender bem o que está sendo feito
2. **Estimativa de capacidade** — Pico de usuários, requests por segundo
3. **Modelagem de dados** — Quais as principais relações, NoSQL, tipo de NoSQL
4. **API Design**
5. **System Design**
6. **Exploração** — Perguntas a serem feitas sobre essa solução, se você pode justificar o que pode explicar

## Requisitos

- **Core Features e domínio** — Entendimento do domínio da aplicação e suas principais funcionalidades
- **Support Features** — Funcionalidades auxiliares que farão com que as funcionalidades principais sejam atendidas
