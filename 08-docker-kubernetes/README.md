## Conceitos Fundamentais Docker

- **Image** vs **Container** — image é o template, container é a instância em execução
- Flags essenciais: `-d` (detached), `-it` (interativo), `--rm` (auto-delete), `-p` (port mapping), `--name`
- `docker exec` vs `docker run` — exec entra num container rodando, run cria novo
- **Volumes** — storage persistente que sobrevive à remoção de containers
- **Networks** — containers na mesma rede se comunicam por nome
